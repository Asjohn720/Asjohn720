## 🧠 My Philosophy

> *"I don't believe learning should be limited to a single field."*

Some projects teach **software architecture**. Some teach **electronics**. Some teach **networking**. Some teach **AI**. 
The goal is always the same: **Understand how things work, then build something useful with that knowledge.**

Rather than focusing on a single domain, I like understanding systems end-to-end — from user interfaces and backend architecture to embedded devices, automation, networking, and AI-powered workflows. I enjoy exploring how software and hardware can work together to solve real-world problems.
